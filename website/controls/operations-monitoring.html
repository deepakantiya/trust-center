Operations and monitoring controls ensure that security events are detected, investigated, and acted upon — and that systems remain resilient and recoverable. Evidence is collected in evidence/monitoring/, evidence/incidents/, and evidence/bcdr/.

Control ID Control Name Implementation SOC 2 ISO 27001 CMMC L1 CMMC L2 Status
AU-2 Event Logging Defined log event taxonomy covering authentication, authorization, data access, configuration changes, and system errors; logged across cloud, application, and endpoint layers. CC7.2 8.15 — AU.2.041 ✅ Implemented
AU-6 Audit Record Review, Analysis & Reporting 24/7 SIEM monitoring with automated alerting; monthly review of alert efficacy and false-positive tuning; anomalous behavior flagged for manual investigation. CC7.2, CC7.3 8.15, 8.17 — AU.2.042 ✅ Implemented
AU-8 Time Stamps NTP synchronization enforced across all systems; logs use UTC timestamps with millisecond precision; clock drift monitored and alerted. CC7.2 8.17 — AU.2.042 ✅ Implemented
AU-9 Protection of Audit Information Logs forwarded to immutable, write-once storage immediately upon generation; ≥90-day hot retention, 1-year cold; access restricted to security team; tampering alerts configured. CC7.2 8.15 — AU.2.042 ✅ Implemented
AU-12 Audit Record Generation All system components generate audit records for defined events; logging agents deployed via IaC and monitored for health; gaps in log coverage trigger automated alerts. CC7.2 8.15 — AU.2.041 ✅ Implemented
SI-4 System Monitoring SIEM with behavioral analytics and correlation rules; EDR on all endpoints with real-time threat detection; network flow monitoring; cloud-native threat detection (GuardDuty / Security Command Center). CC7.2, CC7.3 8.15, 8.16 — AU.2.041 ✅ Implemented
IR-4 Incident Handling Incident response plan with SEV 1–4 severity tiers; P0 triggers immediate response; defined containment, eradication, recovery, and lessons-learned phases; postmortem required within 5 business days for SEV-1/2. CC7.3, CC7.4 5.24, 5.26 — IR.2.092, IR.2.093 ✅ Implemented
IR-6 Incident Reporting 72-hour GDPR/CCPA regulatory notification SLA; customer breach notification drafted by Security + Legal and approved by CEO; internal reporting via security@ and ticketing system. CC7.3, CC2.3 5.24, 6.8 — IR.2.092 ✅ Implemented
IR-8 Incident Response Plan Incident response plan documented, reviewed annually, and tested via tabletop exercises; includes runbooks per incident type, on-call rotation, and pre-staged communication templates. CC7.3 5.24 — IR.2.092 ✅ Implemented
CP-2 Contingency Plan BCDR plan maintained with RTO ≤4h / RPO ≤15min for Tier 1 systems; Business Impact Analysis refreshed annually; DR runbooks per critical system; crisis management roster documented. A1.3 5.29, 5.30 — RE.2.137 ✅ Implemented
CP-9 System Backup Automated encrypted backups for all Tier 1/2 systems; stored in a separate region/account; backups tested quarterly with documented restore verification; retention per data classification schedule. A1.2 8.13 — RE.2.137 ✅ Implemented
CP-10 System Recovery & Reconstitution Annual DR exercise with functional failover test; recovery validated end-to-end; results and gaps tracked; enhanced monitoring for ≥14 days post-recovery. A1.3, CC7.5 5.30 — RE.3.139 ✅ Implemented
CM-6 Configuration Settings Hardening baselines enforced via IaC (Terraform / Ansible); CIS Benchmarks applied to cloud workloads, OS images, and container base images; configuration drift detected and alerted. CC5.2 8.9 — CM.2.061 ✅ Implemented
CM-7 Least Functionality Application allowlisting on managed endpoints; unnecessary network ports and services disabled; container images use minimal base images; unused cloud services and APIs disabled by policy. CC6.8 8.19 — CM.2.064 ✅ Implemented
RA-5 Vulnerability Monitoring & Scanning Continuous infrastructure vulnerability scanning; scan cycle ≤30 days; findings prioritized by CVSS score and exploitability; remediation tracked to closure with SLA enforcement. CC7.1 8.8 — RM.2.141 ✅ Implemented
