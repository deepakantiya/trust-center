Application security controls embed security throughout the software development lifecycle — from threat modeling at design through continuous scanning in CI/CD and annual penetration testing in production. Evidence is collected in evidence/pen-tests/ and evidence/vulnerability-mgmt/.

Control ID Control Name Implementation SOC 2 ISO 27001 CMMC L1 CMMC L2 ISO/SAE 21434 Status
SA-3 System Development Life Cycle Secure SDLC policy mandates security review at design, build, test, and deploy phases; security requirements tracked as acceptance criteria. CC8.1 8.25 — SA.2.068 10.4 ✅ Implemented
SA-8 Security & Privacy Engineering Principles Threat modeling (STRIDE) required for new services and major architecture changes; privacy-by-design review for features handling personal data; secure-by-default configurations. CC8.1 8.25, 8.26 — SA.2.068 10.4 ✅ Implemented
SA-11 Developer Testing & Evaluation SAST (Semgrep / CodeQL), SCA (Dependabot / Snyk), DAST, and secret scanning run on every pull request. Critical/High vulnerabilities in dependencies block merge. CC7.1, CC4.1 8.29 — SA.3.072 11.1 ✅ Implemented
SA-15 Development Process, Standards & Tools Per-language coding standards published; OWASP Top 10 training annually for all developers; pre-commit hooks enforce secret scanning and linting; approved toolchain enforced via CI. CC8.1 8.25, 8.28 — SA.2.068 10.4.2 ✅ Implemented
CM-3 Configuration Change Control All code changes via pull request with required peer review and CI pass; direct pushes to main prohibited; emergency changes retrospectively approved within 48 hours. CC8.1 8.32 — CM.3.068 9.3 ✅ Implemented
CA-8 Penetration Testing Annual external penetration test by qualified third-party firm; scope includes production APIs, authentication flows, and CUI system boundaries. Findings remediated per severity SLAs: Critical 7d, High 30d, Medium 90d. CC4.1, CC7.1 8.29 — SA.3.072 11.1 ✅ Implemented
RA-5 Vulnerability Monitoring & Scanning Continuous vulnerability scanning of infrastructure, containers, and dependencies; scan cycle ≤30 days. Critical ≤7 days / High ≤30 days / Medium ≤90 days remediation SLAs for internet-facing systems. CC7.1 8.8 — RM.2.141 — ✅ Implemented
